HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Headers have 6 levels to them, h1 should only be used once on a webpage
  • Images need a source (typed as "src") and a description (typed as "alt")
  • Resource for HTML: HTML Tutorial

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how mcuh space we want around the content inside an element.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • External CSS style sheets are preferred to create "separation of concerns".
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use. It's based on the idea that each section of code should have its own responsibility.
  • Inline CSS allows to add CSS coding inline with HTML coding, but is infelxible.
  • Internal CSS style sheet puts CSS styles in the "Head" of the HTML sheet but puts all you code together in one file.
  • Class attribues allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • Classs atrtibute can be assigned to the element in HTML, enabling us to freely choose which elements will be assigned the CSS rule
  • Resource for CSS: MDN Web Docs on CSS, CSS Tutorial

Git

  • git status: checks what branch we are currently on
  • git checkout -b "branch-name": creates a new branch and switches to it
  • git pull: fetch and merge any commits from the remote repository/branch
  • git pull origin main: pulls and updates local repository (computer file) the latest update of the main branch from a remote repository (github)
  • git push: uploads content to a remote respository (github)
  • git rm "file": delete the file from project
  • git log: shows all commits in the current branch's history
  • git add -A: adds all files in working branch as it looks now to your next commit (stage)
