HTML

  • HTML stands for Hyper Text Markup Language. It is the standard markup language for creating webpages.
  • HTML consist of a series of elements. Elements tell the browser how to display content.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Header (h) is an element that has 6 levels, h1 should only be used once on a webpage.
  • Image element needs a source (src) and a description (alt).
  • List can be created in unordered (ul) or ordered (ol) fashion with list items (li) nested within.
  • Resource for HTML: HTML Tutorial, HTML Meta, HTML Color Codes

CSS

  • Cascading Style Sheets (or CSS) is the language used to style and HTML document. Describes how HTML elements should be displayed.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
    • External CSS style sheet is a seperate file containing only the CSS code that is linked to the HTML file. This is preferred to create "separation of concerns".
    • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use. Think of it as each section of code should have its own responsibility.
    • Inline CSS allows to add CSS coding inline with HTML coding, but is infelxible.
    • Internal CSS style sheet puts CSS styles in the "Head" of the HTML sheet but puts all you code together in one file.
  • CSS Syntac consists of a selector (points to HTML element to be styled) and declaration block (how HTML element should be styled).
  • Declaration block can contain one or more declasrations separated by semicolons. Each declaration includes CSS property name and value, separated by a colon. Some examples:
    • A very simple example would be "color: red;" where color is the property and red is the value.
    • To align text in a certain fashion we'd use the property "text-align" and the value would indicate alignment (ex: center).
    • Other properties include:
      • A margin indicates how much space we want around the outside of an element.
      • A padding indicates how much space we want around the content inside an element.
      • Height/width are used to set the height and width of an element. Values include auto (default), length (defined in px, cm, etc.), percentage %, and more.
  • The class selector allows us to style any element we choose by assigning the rule to a class attribute with a class selector.
  • Class atrtibute can be assigned to the element in HTML, enabling us to freely choose which elements will be assigned the CSS rule. Ex: Each topic section is classed as "card" which allows for each section to be styled in a box with a shadow.
  • Resource for CSS: MDN Web Docs on CSS, CSS Tutorial

Git

  • Git is a version control system that's responsible for everything GitHub related that happens locally on your computer.
  • Key terms to understand:
    • Repository (or Repo): In short is a project on GitHub. Contains all files and revision histories.
    • Branch: Each repository can contain multiple copies of the same project. Each separate version is a "branch" of the original. Allows for separation of work to be done on same project without running into conflicts and allows for ease of version control.
    • Merging: When done with revisions on a branch, that branch can then be merged into the main branch.
    • Commit: A record of change. Has a unique id for tracking purposes and contains user-defined messages that describes changes made.
    • Issues: Used to help organize yourself within GitHub. Commonly used to track bugs or to-dos.
    • Pull/push: "Pulling" is to pull respository files onto local copies. Pushing is to update repository files with changes made locally.
    • Pull request: Allows for other users to review your "push" from branch before merged into main branch. Common practice for collaborative projects.
  • Below are common Git commands:
    • git status: checks what branch we are currently on
    • git checkout -b "branch-name": creates a new branch and switches to it
    • git pull: fetch and merge any commits from the remote repository/branch
    • git pull origin main: pulls and updates local repository (computer file) with the latest update of the main branch from a remote repository (github)
    • git push: uploads content to a remote respository (github)
    • git rm "file": delete the file from project
    • git log: shows all commits in the current branch's history
    • git add -A: adds all files in working branch as it looks now to your next commit (stage)
  • Resources for GitHub and Git: Intro to GitHub, Git Cheat Sheet, Guide to GitHub

JavaScript

  • JavaScript adds interactivity to a website.
  • Variables are containers that store values. Example: Variable is a box called "Shape", the value Circle is contained in the box "Shape".
  • There are several types of variables:
    • String: A squences of text, must be enclosed in single or double quote marks.
    • Number: Used for numbers. Does not have quotes around them.
    • Boolean: True/False value. True/False are special keywords that don't require quote marks.
    • Array: Strucutre that allows storing mulitple values in a single reference. Example: Variable "Shapes" stores the values Circle, Square, and Triangle.
    • Object: Everything in JavaScript is an object and can be stored in a variable.
  • Conditionals are code strucutres used to test if an expression returns true or not. A very common conditional is the "if...else" statement.
  • Functions defines a block of code to perform a specific task. Functions do not excute automatically, instead you must call a fuction by name in order to execute. This allows the code to be more reasily reusable and ease of control.
  • JavaScript supports control flow statements. Control flow is the order in which a computer executes code in a script.
